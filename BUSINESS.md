# 事業計画：AI面接練習アプリ「Mensetsu Studio」の自動収益化

> 目標：**手元資金1万円 → 1ヶ月以内に「毎月3万円が自動で入る」状態を構築する。**
> 制約：**運用は1日1時間以下。仕組みで回す（労働ではなく資産化）。**

このリポジトリには、その仕組みを成立させるための**課金システム一式**を実装済みです。
本書はビジネス設計・数字の根拠・日次運用フローをまとめたものです。

---

## 1. なぜこのビジネスか（既存資産の活用）

ゼロから何かを作るのではなく、**すでに動いているプロダクト**を収益化します。

このアプリは「AIが面接官になり、音声で深掘り質問 → 5段階評価とレーダーチャートで採点」する
面接練習SaaSです。就活生・転職者という**お金を払う動機が明確な層**が対象で、
すでに「1日15分まで無料」という利用制限ロジックが存在していました。

→ **無料体験はそのまま集客に使い、上限を外す価値に課金する**フリーミアムが最短ルートです。

---

## 2. 収益モデル：フリーミアム × 月額サブスク

| プラン | 価格 | 1日の練習上限 | 役割 |
|--------|------|--------------|------|
| Free | ¥0 | 15分 | 集客・体験（口コミの入口） |
| Premium | **¥980 / 月** | 120分（実質無制限） | 収益の柱 |

サブスクは「一度仕組みを作れば毎月自動で課金される」ため、**3万円/月という継続収入**と相性が最良です。
買い切りや受託は毎月ゼロから売り直す必要があり、自動化・継続性の要件に合いません。

### 損益分岐（3万円の作り方）

```
必要な有料会員数 = 30,000円 ÷ 980円 ≒ 31人
Stripe手数料(約3.6%)を考慮 → 32人で 月¥31,360（手取り 約¥30,230）
```

**有料会員32人**が目標。無料→有料の転換率を保守的に3%とすると、
月に**約1,100人の無料ユーザー**を集めれば到達します（後述の集客で十分射程内）。

### 初期費用（手元1万円の使い道）

| 項目 | 費用 |
|------|------|
| ドメイン（1年） | 約¥1,500 |
| Vercel（フロント） | ¥0（無料枠） |
| Supabase（認証・DB） | ¥0（無料枠） |
| 音声サーバー（Render等） | ¥0〜¥700/月（無料枠 or 最小） |
| Stripe | 初期費用¥0（成果報酬型の手数料のみ） |
| Gemini API | 無料枠 + 従量。会員増加分は売上でカバー |
| **合計** | **約¥2,000〜3,000（1万円の範囲内）** |

固定費がほぼゼロなので、**会員が1人増えるごとに利益が出る**構造です。

## 3. 実装済みの「自動化」システム

このPRで追加したコードにより、**人手を介さずに課金〜権限付与〜解約**まで回ります。

```
ユーザー →[/pricing で「プレミアムを始める」]
        → /api/billing/checkout が Stripe Checkout を生成
        → Stripe 決済画面でカード入力（カード情報は当方に渡らない）
        → 決済成功
        → Stripe Webhook → /api/billing/webhook
        → subscriptions テーブルを plan='premium' に自動更新
        → 音声サーバー(server.ts)が接続時にプランを判定し上限を120分に自動拡張
```

- **解約・支払い変更・領収書**：`/api/billing/portal`（Stripeカスタマーポータル）でユーザーが自己完結。問い合わせ対応ゼロ。
- **支払い失敗**：`invoice.payment_failed` を受けて自動的に free へ降格。
- **権限管理**：すべて `subscriptions` テーブル＋Webhookで自動同期。手動操作は一切不要。

| 追加ファイル | 役割 |
|--------------|------|
| `lib/plan.ts` | プラン・価格・上限の単一の定義 |
| `lib/stripe.ts` | Stripeクライアント |
| `lib/subscription.ts` | 有効プラン判定（フロント用） |
| `lib/supabase/admin.ts` | Webhook用サービスロールクライアント |
| `app/api/billing/checkout/route.ts` | 加入フロー |
| `app/api/billing/portal/route.ts` | 解約・管理フロー |
| `app/api/billing/webhook/route.ts` | **自動化の心臓部**（権限同期） |
| `app/pricing/page.tsx` | 料金ページ |
| `server.ts`（改修） | プラン別の利用上限ゲート |
| `supabase/schema.sql` | `subscriptions` テーブル定義 |
| `app/interview/page.tsx`（改修） | **無料枠到達時のアップグレード導線**（最大の転換点） |
| `lib/articles.ts` ＋ `app/articles/**` | **SEO集客記事システム**（自己PR/逆質問/ガクチカ/頻出質問） |
| `app/sitemap.ts` / `app/robots.ts` / `app/layout.tsx` | SEO基盤（sitemap・robots・メタデータ/OGP） |

## 4. 集客と転換（無料ユーザー約1,100人/月をどう作るか）

転換率3%なら有料32人に届きます。面接・就活はSEOと相性が良く、**一度書けば資産になる**コンテンツで集めます。

> **実装済み**：集客→体験→課金の動線はコードに組み込み済みです。
> - SEO記事システム（`/articles`）＋初期記事4本（自己PR・逆質問・ガクチカ・頻出質問）。各記事末から `/interview`・`/pricing` へ誘導。
> - `sitemap.xml` / `robots.txt` / メタデータ・OGP・構造化データ（Article JSON-LD）。
> - **無料枠到達の瞬間にアップグレードモーダル**（`/interview`）を表示 ＝ 最も成約しやすい転換点。
>
> 運用は「記事を1日1本足す（AI下書き→確認→`lib/articles.ts`に追記）」だけで集客資産が積み上がります。

1. **SEOブログ**（最重要・自動化向き／実装済み）
   面接の頻出質問・回答例・業界別対策などの記事を量産。検索流入は広告費ゼロで積み上がる。
   記事作成はAIで下書き → 人は確認のみ（1日30分の中心作業）。
2. **SNS（X / TikTok）**
   「AI面接官に詰められてみた」系の短尺は拡散しやすい。予約投稿で自動化。
3. **大学キャリアセンター・就活コミュニティ**への無料案内（初動の種まき）。

無料で15分使えるため、ハードルが低く口コミが回りやすいのが構造的な強み。

## 5. 日次運用フロー（1日1時間以下）

| 時間 | 作業 | 自動化度 |
|------|------|----------|
| 5分 | Stripe ダッシュボードで売上・解約を確認 | 確認のみ |
| 5分 | サーバー稼働・エラー監視（Vercel/Render/Supabase） | アラート任せ |
| 30分 | SEO記事を1本（AI下書き→確認→公開） | 半自動 |
| 10分 | SNS予約投稿のストック | 半自動 |
| 10分 | （あれば）ユーザーの声・改善メモ | 任意 |
| **合計** | **約60分** | — |

課金・解約・権限付与・領収書発行は**全自動**。手作業は集客コンテンツと監視だけです。

## 6. セットアップ手順（本番投入）

1. `supabase/schema.sql` を Supabase SQL Editor で実行（`subscriptions` 作成）。
2. Stripe で商品「Premium」を作成し、月額¥980の **Price ID** を取得。
3. Stripe Webhook を `https://<本番ドメイン>/api/billing/webhook` に設定し、
   `checkout.session.completed` / `customer.subscription.*` / `invoice.payment_failed` を購読。署名シークレットを取得。
4. 環境変数を設定（`.env.example` 参照）：
   `STRIPE_SECRET_KEY` / `STRIPE_WEBHOOK_SECRET` / `STRIPE_PRICE_ID` / `NEXT_PUBLIC_SITE_URL` /
   `PREMIUM_DAILY_LIMIT_SECONDS`。
5. デプロイ。`/pricing` から実際に加入 → `/mypage` でプレミアム表示・上限拡張を確認。
